## 1. 项目概述

### 1.1 项目背景

关中智慧校园是一款面向学校管理人员、教师和教职工的微信小程序，旨在对校内特殊学生群体进行系统化、信息化的管理与跟踪，覆盖**六类学生管理**、**四特学生管理**、**在籍不在校管理**、**违纪学生管理**、**校园安全管理**、**安全教育主题管理**、**德育活动管理**、**违禁品管理**、**巡逻打卡管理**九大业务板块。

### 1.2 项目目标

- 建立统一的学生特殊群体信息档案
- 班主任通过**统一标签**提交特殊学生，后端自动归类到六类/四特/在籍不在校
- 实现多层次角色协同管理（管理员 → 校长/副校长 → 德育处/安全办 → 班主任/心理老师/教官）
- 在籍不在校学生触发劝学与家访工作流
- 支持违纪学生从发现→处理→审核→归档的全流程跟踪
- 心理老师对学生进行定期跟踪与谈心谈话记录
- 安全办主任发布学期安全教育计划，班主任每周完成并上传照片反馈
- 违禁品从查获→审核→入柜→领取的全流程管理
- 支持 Excel 批量导入学生基础信息
- 支持数据导出（Excel 统计报表 + PDF 个人档案）

### 1.3 使用人群

| 层级 | 角色 | 类型 | 说明 |
|:---:|------|:---:|------|
| L0 | 管理员（超级管理员） | 个人 | 管理全校组织架构、角色权限、所有数据 |
| L1 | 校长 | 个人 | 查看全校数据，与副校长平级、分工不同 |
| L1 | 德育/安全副校长 | 个人 | 分管德育与安全工作，可为1人或2人 |
| L2 | 德育处主任 | 个人 | 管理部门下属的班主任/心理老师/教官，审核违禁品 |
| L2 | 德育处副主任 | 个人 | 协助主任管理部门，不可做最终处分审核 |
| L2 | 安全办主任 | 个人 | 管理部门下属的安全办成员，发布安全教育计划 |
| L3 | 班主任 | 个人 | 管理所带班级学生，提交特殊学生标签，完成安全教育任务 |
| L3 | 心理老师 | 个人 | 跟踪特异心理/心理类学生 |
| L3 | 教官 | 个人 | 管理违纪学生处理 |
| L3 | 安全办成员 | 个人 | 处理校园安全事务 |

## 2. 组织架构与角色权限

> **核心原则**：所有角色、权限的创建与分配统一由**管理员**在后端管理平台设置，前端小程序仅体现权限效果。

### 2.1 组织架构树

```

│
├── 校长 ─────────────────── 全校数据查看
│
├── 德育/安全副校长 ───────── 分管德育与安全（可合并1人，也可分开2人）
│   │
│   ├── 德育处（部门）
│   │   ├── 德育处主任
│   │   ├── 德育处副主任
│   │   ├── 德育处成员
│   │   ├── 班主任 ───────── 每人绑定1个班级
│   │   ├── 心理老师
│   │   └── 教官
│   │
│   └── 安全办（部门）
│       ├── 安全办主任 ──── 发布安全教育计划
│       └── 安全办成员
│
└── 全校所有数据（管理员可直接查看/操作任何数据）

- 所有权限配置统一在管理员后台完成

## 3. 登录与注册

### 3.1 登录流程

```
微信授权登录 → 绑定手机号 → 选择身份 → 管理员手动审核 → 审核通过 → 进入系统
```

### 3.2 详细步骤

1. **微信授权登录**：调用 `wx.login` 获取 openid，静默登录
2. **绑定手机号**：使用微信 `getPhoneNumber` 快速获取手机号；失败时回退到手动输入 + 短信验证码
3. **选择身份**：用户从可选角色中选择其一（校长 / 德育副校长 / 安全副校长 / 德育处主任 / 安全办主任 / 班主任 / 心理老师 / 教官 / 安全办成员）
   - **管理员**不可自行注册，需在后台预先设置
4. **管理员审核**：提交后等待管理员在后台手动审核通过
5. **绑定班级/年级**（班主任）：审核通过后，选择所带的年级和班级

## 4. 学校组织架构

### 4.1 层级结构

```
学校
 └── 年级（七年级、八年级、九年级、已毕业）
      └── 班级（如：2301班、2401班、2501班）
           └── 学生
```

### 4.2 班级命名规则

采用**入学年份 + 班级序号**，如 `2301班` = 2023 年入学的 01 班。

### 4.3 学年周期

- **入学**：每年 9 月
- **毕业**：每年 7 月
- 毕业后学生数据保留，管理员仍可查看

### 4.4 年级自动升级

> 每年 **8 月**，系统自动执行年级升级，无需手动操作。

| 升级前 | 升级后 | 说明 |
|------|------|------|
| 七年级 | 八年级 | 正常升级 |
| 八年级 | 九年级 | 正常升级 |
| 九年级 | 已毕业 | 标记毕业，数据保留可查 |

- 升级时间：每年 8 月 1 日（管理员可配置具体日期）
- 升级后班级命名不变（如 2301 班始终为 2301 班），仅年级归属变更
- 已毕业年级的学生所有标签和记录保留，仍可查询和导出
- 管理员可在后台**手动触发**年级升级（用于测试或特殊情况）

### 4.5 学期周次计算

> 管理员在后台设置**本学期第一周的起始日期**，系统自动计算后续周次。此周次用于安全教育周计划等周次相关功能。

| 配置项 | 说明 |
|------|------|
| 第一周起始日期 | 管理员选择一个日期（通常为开学第一周的周一） |
| 自动计算 | 以第一周起始日期为基准，每7天递增一周（第1周、第2周...第N周） |
| 当前周 | 系统根据当前日期自动判定属于第几周 |
| 学期结束 | 管理员可设置学期总周数（如20周），超出后标记为"学期已结束" |

> 此周次计算主要用于：
> - 安全教育主题管理的周次定位（第11章）
> - 班主任收到的"第X周安全教育主题"任务
> - 逾期判断：超过当周未提交反馈即为逾期

### 4.6 关键约束

- 一个班主任负责 **1 个班级**
- 一个学生可同时拥有多个标签（标签可归属于六类/四特/在籍不在校中的一个或多个体系）
- 学生基础信息包含：班级、姓名、性别、身份证号码、家庭住址、父亲姓名、父亲联系电话、母亲姓名、母亲联系电话、备注信息（父亲姓名、父亲联系电话、母亲姓名、母亲联系电话，这四两组信息，至少要一组，可以空一组）

## 5. 统一标签提交设计

## 6. 六类学生管理

> **说明**：六类与四特、在籍不在校是三套独立的分类体系。后端根据标签映射表自动将学生归入六类。

### 6.1 分类定义

| 序号 | 类别 | 包含情形（通过标签自动归类） |
|:---:|------|---------|
| 1 | **家庭类** | 单亲家庭、孤儿、离异家庭、寄住亲戚家、留守儿童、家庭监管严重缺失、经济特别困难 |
| 2 | **网瘾类** | 存在网瘾、迷恋手机等问题 |
| 3 | **心理类** | 心理筛查有心理问题、性格孤僻、自卑、情绪反常、成绩突然下滑、精神恍惚、无故旷课 |
| 4 | **休学辍学类** | 休学学生、义务教育阶段疑似辍学 |
| 5 | **身体类** | 身体残疾、特殊疾病、特异体质、存在突发性危险的疾病 |
| 6 | **其他类** | 就读工读学校、有违纪违法案底、学业负担过重/压力过大、存在其他问题和困境 |

> **跨体系说明**：部分标签同时归属于在籍不在校（如休学、疑似辍学、工读学校），此类学生会在两个体系中同时显示，确保不同管理视角不遗漏。

### 6.2 管理功能

- 班主任通过统一标签入口提交，后端自动归入六类
- 管理员/德育处主任可在后台复核和调整归类
- 每条记录包含：学生信息、所属标签、所属六类类别、具体情形描述、添加日期、更新日期
- 支持**按类别筛选**、**按班级筛选**
- 校长、副校长可全校查看，管理员可全校管理

### 6.3 退出机制

- 学生毕业后数据保留，管理员仍可查看
- 标签可更新但不可直接删除（保留历史痕迹）

## 8. 在籍不在校管理

> **说明**：在籍不在校是独立于六类、四特的第三套分类体系，聚焦学生"人不在学校"的状态，并配有**劝学**和**家访**工作流。

### 8.1 分类定义

| 序号 | 情形 | 说明 |
|:---:|------|------|
| 1 | **长时间请假** | 一次请假超过一周（不含正常病假/事假） |
| 2 | **疑似辍学** | 无故连续多日不到校，有辍学风险 |
| 3 | **休学** | 已正式办理休学手续 |
| 4 | **工读学校** | 就读于工读学校 |
| 5 | **特殊学校** | 就读于特殊教育学校 |
| 6 | **借读** | 学籍在本校但在其他学校借读 |

### 8.2 劝学工作流

> **触发条件**：学生被标记为"长时间请假"或"疑似辍学"，且超过 **7 天** 未恢复正常在校状态。

```
标记为"长时间请假"/"疑似辍学"
        ↓
   超过 7 天未恢复
        ↓
   系统自动/手动触发劝学任务
        ↓
   班主任/德育处主任记录劝学情况
        ↓
   劝学有结果？──→ 恢复在校 → 更新标签状态
        ↓ 未恢复
   继续跟踪 ──→ 启动家访工作流
```

> **提醒机制**：劝学超时未处理时，系统提醒**德育处主任**，由德育处主任去提醒对应班主任开展家访。

### 8.3 家访工作流

> **要求**：每学期至少 **2 次**家访。

| 字段 | 说明 |
|------|------|
| 家访日期 | 日期 |
| 家访人员 | 班主任 / 德育处主任 / 其他 |
| 家访方式 | 上门 / 电话 / 线上 |
| 家访对象 | 家长姓名 |
| 家访内容 | 文字描述 |
| 学生近况 | 文字描述 |
| 劝返结果 | 已返校 / 仍未返校 / 家长拒绝 |
| 后续计划 | 文字描述 |
| 附件 | 图片（如家访照片） |
| 记录时间 | 创建/更新时间 |

### 8.4 劝学记录字段

| 字段 | 说明 |
|------|------|
| 劝学日期 | 日期 |
| 劝学方式 | 电话 / 上门 / 线上 |
| 劝学对象 | 学生本人 / 家长 / 两者 |
| 劝学内容 | 文字描述 |
| 劝学结果 | 已返校 / 承诺返校 / 仍未返校 |
| 后续计划 | 文字描述 |
| 记录人 | 班主任或德育处主任 |
| 记录时间 | 创建/更新时间 |

### 8.5 在籍不在校状态流转

```
正常在校
  → 被标记为"长时间请假"/"疑似辍学" → 跟踪中
  → 超过7天 → 劝学中
  → 启动家访 → 家访中
  → 学生返校 → 恢复正常在校（标签保留，状态改为"已恢复"）
  → 学生正式休学/转学 → 登记为"休学"/"特殊学校"等（状态固定）
```

## 9. 违纪学生管理

### 9.1 违纪类型

#### 普通违纪

日常轻微违纪行为，记录后可进行口头教育、书面检讨等处理。支持**文字描述**和**拍照上传**两种记录方式。

#### 处分等级

| 级别 | 处分类型 | 说明 |
|:---:|------|------|
| 1 | 警告 | 最轻处分 |
| 2 | 严重警告 | — |
| 3 | 记过 | — |
| 4 | 留校察看 | 最严重处分（不含开除） |

### 9.2 惩戒措施

| 惩戒类型 | 说明 |
|------|------|
| 停宿 | 暂停住宿资格，需指定天数 |
| 停学 | 暂停上学，需指定天数 |

### 9.3 管理流程

```
发现违纪 → 班主任/教官提交违纪记录 → 录入违纪详情 → 处理过程跟踪
                                                              ↓
                                         归档 ← 管理员最终审核 ← 处分决定
```

### 9.4 处分降级与撤销

- **处分降级**：提交文字 + 图片依据，将处分等级降低（如留校察看 → 记过）
- **处分撤销**：提交文字 + 图片依据，撤销处分
- **关键规则**：降级/撤销操作**不直接删除**原记录，仅修改状态标记，历史记录可查询
- 处分降级和撤销需**管理员审核**

### 9.5 违纪记录数据结构

| 字段 | 说明 |
|------|------|
| 学生信息 | 关联学生基础信息 |
| 违纪类型 | 普通违纪 / 处分（含级别） |
| 违纪日期 | 发生日期 |
| 违纪详情 | 文字描述 + 拍照图片 |
| 惩戒类型 | 停宿（天数）/ 停学（天数）/ 无 |
| 处理过程 | 教官/班主任填写的处理经过 |
| 处理结果 | 最终处理结果 |
| 处分决定 | 处分等级及依据 |
| 处分状态 | 生效中 / 已降级 / 已撤销 |
| 处理状态 | 处理中 / 待审核 / 已结案 |
| 记录人 | 班主任或教官 |
| 记录时间 | 创建/更新时间 |

## 11. 安全教育主题管理

> **核心场景**：开学第一周，安全办主任发布整个学期的每周安全教育主题计划，所有班主任收到任务，每周完成安全教育后上传照片反馈。

### 11.1 整体流程

```
开学第一周
     ↓
安全办主任创建学期安全教育计划
     ↓
逐周设置：教育主题 + 教育内容 + 课件（pptx / ppt / 希沃课件）
     ↓
发布 → 全体班主任收到每周任务
     ↓
班主任每周开展安全教育 → 上传1-3张照片 → 提交完成
     ↓
安全办主任查看各班级完成情况
```

### 11.2 发布学期计划（安全办主任）

#### 创建计划

| 字段 | 说明 | 必填 |
|------|------|:---:|
| 学期 | 如：2026-2027学年第一学期 | ✅ |

| 计划名称 | 如：2026年秋季安全教育周主题计划 | — |

| 字段 | 说明 | 必填 |

```

## 15. 各角色详细需求

### 15.1 管理员（L0 - 超级管理员）

- **组织架构管理**：创建/编辑/删除年级、班级、部门（德育处/安全办）；**设置学期第一周起始日期**；**手动触发年级升级**
- **打卡位置管理**：预设巡逻打卡的楼栋和楼层
- **角色与权限管理**：为所有用户设置角色、分配部门、配置权限
- **标签映射管理**：维护统一标签→六类/四特/在籍不在校的映射关系
- **违禁品类型管理**：维护违禁品类型列表（增删改）
- **用户审核**：审核新注册用户的身份申请
- **学生信息管理**：
  - 手动录入/编辑/查看学生基础信息
  - **Excel 批量导入**：上传 Excel 导入学生信息
  - 导入字段：班级、姓名、性别、身份证号码、家庭住址、家长姓名、家长联系方式
  - **身份证号脱敏显示**：列表页仅显示后 4 位；导出时使用完整数据
- **全量数据管理**：可查看和操作所有业务板块数据
- **处理结案权限**：只有管理员可将违纪/安全事件处理状态改为"已完结"
- **在籍不在校管理**：可查看所有在籍不在校学生的劝学/家访记录
- **数据导出**：
  - **Excel 格式**：统计数据（六类生、四特、在籍不在校、违纪、安全事件、安全教育完成情况、违禁品）
  - **PDF 格式**：单个学生完整档案（个人信息 + 标签 + 违纪记录 + 谈心谈话 + 劝学家访 + 违禁品记录）
- **操作日志查看**：记录谁在什么时间做了什么操作
- **所有角色功能的超集**：管理员可以执行任何其他角色的操作

### 15.2 校长（L1）

- **首页看板**：全校特殊学生总数、各类别人数统计、违纪统计、在籍不在校统计、安全教育完成率、违禁品统计
- **全校数据查看**：按年级 → 班级查看各班各业务板块数据
- **安全教育查看**：查看每周各班级安全教育完成情况
- **筛选查询**：按类别、年级、班级、日期范围筛选
- **安全事件总览**：查看安全办提交的安全事件统计
- **权限**：**只读 **，不可新增/修改/删除数据
- **分管领域**：侧重教学学术相关数据

### 15.3 德育/安全副校长（L1）

- **与校长平级**，权限相同（只读 ），但**分管领域不同**
- **德育相关**：六类学生数据、四特学生数据、在籍不在校数据、违纪数据、违禁品数据
- **安全相关**：安全办提交的安全事件数据、安全教育主题及完成情况
- **筛选查询**：同校长
- **可为1人或2人**：
  - 若为1人：勾选德育 + 安全两个分管领域
  - 若为2人：德育副校长管六类/四特/在籍不在校/违纪/违禁品，安全副校长管安全办/安全教育

### 15.4 德育处主任（L2）

- **部门管理**：查看德育处下属所有人员的数据
  - 所有班主任提交的特殊学生标签记录
  - 所有教官提交的违纪记录
  - 所有心理老师提交的谈心谈话记录
  - 所有在籍不在校学生的劝学/家访记录
- **数据编辑**：可编辑下属提交的记录
- **数据复核**：可复核和调整后端自动归类结果
- **Excel 导入**：可批量导入学生信息
- **数据导出**：可导出德育处相关数据
- **违禁品审核**：审核所有违禁品提交，通过后上传入柜照片
- **违禁品领取**：可处理违禁品领取（与班主任均可操作）
- **自主提交安全教育活动**：可自行提交安全教育主题活动记录（主题 + 简要概括 + 1-3张佐证照片）
- **提交德育活动**：可自行提交德育主题活动记录（主题 + 简要概括 + 1-3张佐证照片）
- **查看巡逻记录**：查看所有教官的巡逻打卡记录，支持筛选、统计、导出Excel、查看详情
- **做违纪处分审核/结案** 德育处主任有**唯一审核权限**

### 15.5 安全办主任（L2）

- **部门管理**：查看安全办下属成员提交的安全事件记录
- **数据编辑**：可编辑下属提交的安全事件
- **数据导出**：可导出安全办相关数据
- **安全教育计划管理（核心功能）**：
  - **创建学期计划**：开学第一周创建整个学期的每周安全教育主题
  - **逐周设置**：每周的教育主题、教育大致内容、课件（pptx/ppt/希沃课件）
  - **发布与修改**：可一次性发布完整学期，也可逐周补充；已发布内容可修改
  - **查看完成情况**：
    - 按周次查看各班级完成率
    - 按班级查看整学期完成情况
    - 查看每期上传的照片反馈
  - **统计导出**：导出安全教育完成情况统计表（Excel）
- **查看巡逻记录**：查看所有教官的巡逻打卡记录，支持筛选、统计、导出Excel、查看详情
- **自主提交安全教育活动**：可自行提交安全教育主题活动记录（主题 + 简要概括 + 1-3张佐证照片）
- **不可做安全事件最终结案**（仅管理员有此权限）

### 15.6 班主任（L3）

- **我的班级**：查看所带班级学生列表
- **提交特殊学生**：通过**统一标签**勾选方式为本班学生提交情形标签
- **在籍不在校管理**：
  - 查看本班在籍不在校学生
  - 提交**劝学记录**
  - 提交**家访记录**（每学期2次）
- **违纪管理**：
  - 提交本班学生违纪记录（文字 + 图片）
  - 查看违纪处理进度
- **安全教育任务（核心功能）**：
  - 接收每周安全教育主题任务（查看主题、内容、下载课件）
  - 完成安全教育后**上传1-3张照片**反馈
  - 查看本班安全教育完成历史记录
  - 未完成/逾期提醒
- **提交德育活动**：可自行提交德育主题活动记录（主题 + 简要概括 + 1-3张佐证照片）
- **违禁品管理**：
  - 可对本班学生**发起违禁品记录**
  - 可**处理本班学生违禁品领取**（拍摄手持照片）
- **学生档案**：查看本班每个学生的完整标签与记录
- **数据范围**：仅限自己所带班级
- 可被**德育处主任**查看和管理

### 15.7 心理老师（L3）

- **学生列表**：查看标记为"特异心理"（四特）+ "心理类"（六类）的学生，**仅限心理类数据**
- **跟踪记录**：
  - 新增/编辑谈心谈话记录（文字 + 图片）
  - 查看跟踪时间线
- **状态更新**：更新学生心理状态评估
- **提醒功能**：设置下次跟进日期，到期提醒
- **违禁品**：可发起违禁品记录
- 可被**德育处主任**查看和管理

### 15.8 教官（L3）

- **违纪学生列表**：查看全校违纪学生
- **处理过程记录**：
  - 填写处理经过（文字 + 图片附件）
  - 更新处理进展
- **处理结果**：填写处理结果
- **违禁品**：可发起违禁品记录
- **巡逻打卡（核心功能）**：
  - 每天不同时段外出巡逻时打卡
  - 打卡时间系统自动记录，位置从预设楼栋+楼层中选择
  - 填写巡逻情况，可选上传照片佐证
  - 查看自己的巡逻打卡历史记录
- **权限约束**：
  - 可提交违纪记录和处理过程
  - **不可**自行将处理状态改为"已完结"（仅德育处主任有此权限）
  - **不可**做处分最终审核（仅德育处主任有此权限）
- 可被**德育处主任**查看和管理

### 15.9 安全办成员（L3）

- **安全事件记录**：
  - 新增/编辑安全事件（文字 + 图片）
  - 更新处理过程
- **安全隐患排查**：提交安全隐患并跟踪整改
- **处理结果**：填写处理结果
- **违禁品**：可发起违禁品记录
- **权限约束**：
  - 可提交和处理安全事件
  - **不可**自行将事件状态改为"已完结"（仅安全办主任有此权限）
- 可被**安全办主任**查看和管理

## 16. 消息通知机制

### 16.1 通知方案

| 阶段 | 方案 | 说明 |
|------|------|------|
| **一期（当前）** | 小程序长期订阅消息 | 教育行业有专属长期订阅模板，用户授权一次即可持续接收通知 |
| **二期（后续）** | + 微信服务号模板消息 | 开通服务号后，覆盖更多通知场景，无需每次订阅 |

### 16.2 一期可覆盖的通知场景

| 触发场景 | 通知对象 | 说明 |
|------|------|------|
| 班主任提交特殊学生标签 | 德育处主任 | 提醒有新的特殊学生记录待查看 |
| 学生被标记为"长时间请假"/"疑似辍学"超7天 | 班主任、德育处主任 | 提醒需启动劝学工作 |
| 劝学/家访记录提交 | 德育处主任 | 提醒有新的劝学/家访记录 |
| 家访次数不足 | 班主任 | 提醒本学期家访次数未达标（<2次） |
| 班主任/教官提交违纪记录 | 管理员、德育处主任 | 提醒有新的违纪记录待审核 |
| 教官更新违纪处理 | 班主任 | 提醒该生所在班级班主任了解处理进展 |
| 德育处主任审核通过处分 | 班主任 + 教官 | 通知处分审核结果 |
| 心理老师跟进提醒到期 | 心理老师 | 提醒对某学生进行定期跟进 |
| 安全办成员提交安全事件 | 管理员、安全办主任 | 提醒有新的安全事件待查看 |
| **安全办主任发布新周主题** | 全体班主任 | 提醒新一周安全教育主题已发布 |
| **安全办主任修改已发布主题** | 全体班主任 | 提醒安全教育主题有更新 |
| **班主任安全教育逾期未完成** | 班主任 | 提醒及时完成安全教育并上传反馈 |
| **安全教育完成率低于阈值** | 安全办主任 | 提醒关注未完成班级 |
| 违禁品提交待审核 | 德育处主任 | 提醒有新的违禁品记录待审核 |
| 违禁品审核通过 | 发起人 | 通知审核结果，提醒存入保险柜 |
| 违禁品审核驳回 | 发起人 | 通知驳回原因，可修改重提 |
| 违禁品已入柜 | 班主任 | 通知该生班主任违禁品已存档 |
| 新用户注册申请 | 管理员 | 提醒有新的注册申请待审核 |

### 16.3 服务号的必要性

**当前阶段不需要服务号**。小程序长期订阅消息已可覆盖上述通知场景。服务号的优势在于无需用户每次订阅、支持离线用户触达、消息类型更灵活。

建议：**一期先用小程序订阅消息上线，等服务号开通后再升级通知能力**。
